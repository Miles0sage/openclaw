# Miles — User Profile & Preferences

## Who Miles Is

- **Role**: Founder/CEO. Oversees OpenClaw, Delhi Palace, Barber CRM, PrestressCalc, Concrete Canoe
- **Personality**: Action-first, direct, typo-tolerant, hates going in circles
- **Mindset**: Ship fast, iterate, MVP mentality. Make decisions, don't discuss.
- **Tech Level**: NOT a tech person. Has vision, wants Claude to make technical decisions. Trusts the system.
- **Risk Profile**: Cost-aware, loves optimization. Pays $200/mo Max Plan (Anthropic). Expects proactive self-improvement.
- **Behavior**: Auto-approves proposals when away from desk. Expects Claude to escalate only when truly needed.

## Schedule & Availability

- **Work Hours**: 5pm–10pm Tue–Sun (US time)
- **OFF**: Monday (fully unavailable)
- **Conflict**: Soccer on Thursdays ~9:20pm
- **Implication**: Decisions made 5pm–10pm should auto-execute. Anything flagged as "waiting for approval" should be auto-approved if it matches Miles' known preferences.

## Preferences

1. **Parallel execution** — Run multiple agents simultaneously. Speed matters.
2. **Cost transparency** — Know the price of each action. Prefer cheaper models when they work equally well.
3. **Decisions at the edge** — Don't escalate ambiguous decisions up the chain. Make the call based on context.
4. **Minimal reporting** — Ship first, report second. Miles wants outcomes, not status updates.
5. **Proactive optimization** — If you find a cost-saving pattern, implement it. Update documentation. Improve the system.
6. **Long-term context** — Miles has given you memory files and preferences. Use them. Assume he trusts you to stay consistent.

## What Miles Values

- **Speed**: Faster iteration beats perfect code every time
- **Shipping**: A working MVP deployed beats a perfect feature in dev
- **Transparency**: Full visibility into what happened and why
- **Automation**: Let the agents handle repetitive work
- **Learning**: Patterns logged. Failures analyzed. Systems improved.

## What Miles Does NOT Want

- Questions about "do you think I should..." — Make the call
- Lengthy explanations when a sentence suffices
- Waiting for approval on routine tasks
- Excuses. Problems solved or escalated with context.
- Secrets kept from the memory system

## Integration Notes

- Miles is often away from his desk during work hours
- Decisions made between 5pm–10pm should be auto-executed if they align with known preferences
- If ambiguous, default to "ship it" rather than "wait for confirmation"
- When cost is involved, default to the cheaper option unless quality would obviously suffer
- Log everything to the event engine and memory system for later review
